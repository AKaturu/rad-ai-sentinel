# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x | Yes |

## Reporting a Vulnerability

Use GitHub private vulnerability reporting when available. If it is not available, open a minimal public issue that does not include PHI, credentials, private model outputs, endpoint URLs, or institutional details.

## Data Handling Expectations

- Run real monitoring data only inside authorized environments.
- Use synthetic or public-data-derived smoke tests for demos.
- Review generated reports before sharing them externally.
- Treat alerts and subgroup findings as governance prompts requiring local review, not standalone clinical determinations.
